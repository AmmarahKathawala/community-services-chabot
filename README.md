# Community Services Chatbot

A reference implementation of a **rule-based chatbot** designed to support **routine community care service requests**.

The system models real-world service workflows by:
- collecting structured intake information
- applying safety-aware triage rules
- generating clear handover summaries suitable for service teams

This project is intended as a **portfolio and demonstration project** and emphasises explainable decision logic over black-box automation.

---

## Key Features
- Structured intake for routine requests
- Known vs new patient branching logic
- Safety screening with urgent escalation
- Rule-based decision logic (explainable)
- Auto-generated handover summaries (JSON + readable text)

---

## Tech Stack
- Python
- Streamlit
- Rule-based decision logic

---

## Project Status
This is an active, evolving project.
Planned extensions include:
- Additional request types (e.g. catheter care, occupational therapy, physiotherapy)
- Basic analytics and audit logging
- Improved conversational UI
